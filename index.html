Designing and debugging experience

When an agent software behaves in an unanticipated way, a developer must debug the source code to find the problem. Fault localization necessitates a grasp of how code interacts with observed agent behavior. The major goal of this study is to create a source-level debugger that allows a cognitive agent program to be executed in a single step. Cognitive agents go through a decision cycle in which they process information and use their beliefs and goals to make a decision. Current state-of-the-art debuggers for agent programs give insight into how agent behavior arises from this cycle, but not so much into how it links to program code. The process of finding, finding, and resolving errors in a computer program is known as debugging. Debugging a program takes up a significant portion of a programmer's time.
